# @指令 系统：自然语言高级命令终端

**文档版本：** 1.2.0
**实现时间：** 2026-03-31
**分支：** xxbb

---

## 1. 概述

### 1.1 目的与愿景

**目的：** 从"AI 盲猜 (Passive)" 到 "人类确权 (Human-in-the-loop)"

VCPChat `@指令` 系统旨在为聊天窗口引入**确定性的调度控制权**，将自然语言输入转化为精确的工具调用。

### 1.2 核心功能

1. **极速唤起与级联补全：** 通过 `@` 瞬间唤起系统已挂载的插件列表，结合 `/` 唤起子命令，全程键盘操作。
2. **自然语言提示注入：** 用户显式指定目标插件（如 `@TavilySearch/TavilySearch`），后端从 Registry 获取工具 schema，注入自然语言提示，让 Agent LLM 自行决定如何调用工具。
3. **零侵入架构底座：** 完全复用现有的 VCP 循环（`ToolCallParser` 与 `ToolExecutor`），后端仅通过"消息内容替换"实现路由提示，不破坏正常对话流程。

## 2. 架构设计

### 2.1 数据流图

```
用户输入 @TavilySearch/TavilySearch 帮我搜github热点
        ↓
[前端] 检测 @ 触发 → 模糊搜索插件命令 → 用户选择/补全
        ↓
[前端] 发送 vcp_force_route 请求到后端
        ↓
[后端] chatCompletionHandler.handle() 拦截 vcp_force_route
        ↓
[后端] 从 Registry 读取工具 description + example
        ↓
[后端] 注入自然语言提示（让 Agent 知道如何调用该工具）
        ↓
[后端] 调用 Agent LLM → Agent 自行决定输出 <<<[TOOL_REQUEST]>>> 块
        ↓
[后端] VCP Loop 执行工具 → Agent 解读结果
        ↓
[后端] 返回 Agent 解读后的回复
```

### 2.2 关键文件

| 层级 | 文件 | 职责 |
|------|------|------|
| 前端 | `VCPChat-main/modules/inputEnhancer.js` | @命令补全、模糊搜索、命令历史 |
| 前端 | `VCPChat-main/modules/vcpClient.js` | vcp_force_route 检测与请求 |
| 前端 | `VCPChat-main/modules/messageRenderer.js` | 执行状态 UI 更新 |
| 前端 | `VCPChat-main/renderer.js` | WebSocket 状态事件处理 |
| 后端 | `VCPToolBox_new/server.js` | `/v1/plugins/registry` 路由、REGISTRY_UPDATED 广播 |
| 后端 | `VCPToolBox_new/Plugin.js` | `getPluginRegistry()` 方法 |
| 后端 | `VCPToolBox_new/modules/chatCompletionHandler.js` | vcp_force_route 拦截与强制路由 |
| 后端 | `VCPToolBox_new/WebSocketServer.js` | `broadcastVCPInfo()` 状态广播 |

### 2.3 自然语言提示注入

**目的：** 将 `@指令` 转化为自然语言提示，让 Agent LLM 自行决定如何调用工具，完全依赖 LLM 的理解能力，不再有 Fallback。

**动态 Schema 注入流程：**

1. 后端从 `pluginManager.getPluginRegistry()` 读取目标插件和命令的 description + example
2. 根据 `command` 是否为空，构造不同格式的自然语言提示
3. 替换用户消息内容，让 Agent LLM 看到工具说明后自行决定

**情况 1：只指定了插件名（command 为空）**

注入插件完整描述 + 所有命令列表，让 Agent 帮用户选择用哪个命令：

```
用户要求：帮我搜索 github 上的热门项目

请使用 TavilySearch 插件完成此任务。

插件说明：使用 Tavily API 进行高级网络搜索。AI可以指定搜索查询、主题、最大结果数，并可选择包含原始内容、图片链接及描述，以及设定搜索时间范围。

可用命令：
1. TavilySearch - 高级网络搜索，支持 query, topic, max_results 等参数

请根据用户需求选择合适的命令执行。
```

**情况 2：同时指定了插件名和命令（command 有值）**

注入精确的命令描述和使用示例：

```
用户要求：帮我搜索 github 上的热门项目

请使用 TavilySearch 的 TavilySearch 命令完成此任务。

命令说明：调用此工具使用 Tavily API 进行高级网络搜索...
- query (必需): 搜索的关键词或问题
- topic (可选): 搜索的主题，默认为 'general'
- max_results (可选): 返回的最大结果数量

请根据用户需求填充参数并调用该命令。
```

**后续流程：**
- Agent LLM 看到提示后，自行决定是否输出 `<<<[TOOL_REQUEST]>>>` 块
- 如果输出 → VCP Loop 正常执行工具 → Agent 解读结果 → 返回
- 如果不输出 → Agent 直接返回文字回复（无 Fallback）

## 3. API 规范

### 3.1 插件注册表接口

**端点：** `GET /v1/plugins/registry`

**响应格式：**
```json
{
  "timestamp": 1743468000000,
  "plugins": [
    {
      "name": "ServerFileOperator",
      "displayName": "VCP服务器文件操作器",
      "pluginType": "synchronous",
      "description": "VCP服务器专用的一个强大的文件系统操作插件...",
      "capabilities": {
        "invocationCommands": [
          {
            "command": "ReadFile",
            "description": "功能: 读取指定绝对路径下文件的内容...",
            "example": "<<<[TOOL_REQUEST]>>>\ntool_name:「始」ServerFileOperator「末」..."
          }
        ]
      }
    }
  ]
}
```

### 3.2 强制路由请求

**端点：** `POST /v1/chat/completions` (同一端点，通过 `vcp_force_route` 字段区分)

**请求体：**
```json
{
  "messages": [...],
  "vcp_force_route": {
    "plugin": "OpenCli",
    "command": "generate",
    "user_input": "帮我写个Python贪吃蛇"
  },
  "model": "gpt-4",
  "stream": true
}
```

**响应格式（v1.6.0）：**

v1.6.0 不再返回特殊的 `vcp_force_route_result` 类型。后端注入提示后，响应由 Agent LLM 的正常流程决定（可能经过 VCP Loop 执行工具），最终返回 Agent 的解读性回复。

### 3.3 WebSocket 状态广播

**通道：** VCPInfo (`/vcpinfo/VCP_Key=xxx`)

**消息格式：**
```json
{
  "type": "force_route_status",
  "status": "executing",
  "plugin": "OpenCli",
  "command": "generate",
  "messageId": "msg_1234567890"
}
```

**status 可选值：**
- `executing` - LLM 解析中或工具执行中
- `success` - 执行成功
- `failed` - 执行失败（LLM 解析失败或工具执行异常）

## 4. 前端实现

### 4.1 @指令 检测与补全

**文件：** `VCPChat-main/modules/inputEnhancer.js`

**流程：**
1. 监听 `messageInput` 的 `input` 事件
2. 检测 `@` 字符，提取查询词
3. 若查询词**不包含** `/`，调用 `searchPlugins()` 搜索插件名
4. 若查询词**包含** `/`，调用 `searchPluginCommands()` 搜索该插件下的命令
5. 显示 `cmd-suggestion-popup` 弹出层

**两阶段搜索：**
- **阶段1：** 输入 `@Op` → 搜索插件名 → 显示插件列表（带左边框标识）
- **阶段2：** 选择插件后输入 `@AgentDream/` → 搜索该插件下的命令 → 显示命令列表

**模糊匹配算法：**
- 支持插件名和命令名的模糊匹配
- 匹配分数 = (插件匹配度 + 命令匹配度) / 2
- 按分数降序返回 Top 10

**命令历史：**
- 发送含 `@` 的消息时调用 `recordCommandForHistory()`
- ArrowUp 在空输入框时召回历史命令
- 最多保存 20 条历史记录

### 4.2 vcp_force_route 检测

**文件：** `VCPChat-main/modules/vcpClient.js`

**检测正则：**
```javascript
/@([\w\u4e00-\u9fa5\-]+)\/([\w\u4e00-\u9fa5\-]+)\s+(.+)/
```

**示例输入：** `@OpenCli/generate 帮我写个Python贪吃蛇`

**解析结果：**
```javascript
{
  plugin: "OpenCli",
  command: "generate",
  user_input: "帮我写个Python贪吃蛇"
}
```

### 4.3 执行状态 UI

**文件：** `VCPChat-main/modules/messageRenderer.js`

**状态指示器样式：**
| 状态 | 样式 |
|------|------|
| executing | 黄色背景 + 呼吸动画 |
| success | 绿色背景 + ✓ 图标 |
| failed | 红色背景 + ✗ 图标 + 错误信息 |

## 5. 后端实现

### 5.1 强制路由拦截

**文件：** `VCPToolBox_new/modules/chatCompletionHandler.js`

**注入的系统提示词：**
```
你是一个精确的参数提取器。用户要求你调用工具 <plugin>{plugin}</plugin> 的 <command>{command}</command> 命令。

【强制指令】
1. 严格禁止输出任何对话性文字
2. 严格禁止进行任何解释或评论
3. 你必须根据用户的自然语言输入，提取参数并直接输出工具调用块
4. 只输出下方指定的工具调用格式，不要添加任何其他内容

【输入隔离】
<user_input>{user_input}</user_input>

【输出格式】
<<<[TOOL_REQUEST]>>>
tool_name:「始」{plugin}「末」,
command:「始」{command}「末」,
[参数名]:「始」[参数值]「末」
<<<[END_TOOL_REQUEST]>>>
```

### 5.2 熔断策略

**触发条件：**
1. LLM 响应不包含有效的 `<<<[TOOL_REQUEST]>>>...<<<[END_TOOL_REQUEST]>>>` 块
2. LLM API 调用失败（HTTP 非 200）
3. 工具执行抛出异常

**处理方式：**
- 返回明确的错误信息
- 不降级为普通聊天回复
- 通过 WebSocket 广播 `failed` 状态

### 5.3 插件注册表热更新

**触发时机：**
- `pluginManager.loadPlugins()` 完成时
- `pluginManager.onPluginUpdated()` 触发时

**广播内容：**
```json
{
  "type": "REGISTRY_UPDATED",
  "timestamp": 1743468000000
}
```

## 6. 使用示例

### 6.1 输入阶段

用户在输入框中输入 `@Op`：
- 前端自动弹出插件命令建议列表
- 用户选择 `OpenCli/generate`
- 输入框内容变为 `@OpenCli/generate `

### 6.2 发送阶段

用户补充参数并发送：
```
@OpenCli/generate 帮我写个Python贪吃蛇
```

前端检测到 `@指令` 格式，构造请求：
```json
{
  "vcp_force_route": {
    "plugin": "OpenCli",
    "command": "generate",
    "user_input": "帮我写个Python贪吃蛇"
  }
}
```

### 6.3 执行阶段

1. 后端收到请求，注入系统提示词
2. LLM 解析参数并返回工具调用块
3. 后端解析并执行工具
4. WebSocket 推送执行状态到前端
5. 前端显示呼吸灯动画

### 6.4 完成阶段

执行成功后，前端显示绿色成功状态，用户看到生成的文件内容或执行结果。

## 7. 已知限制

1. **输入格式要求：** 当前仅支持 `@plugin/command 参数` 格式
2. **参数解析依赖 LLM：** 参数提取的准确性取决于上游模型能力
3. **无多命令支持：** 单次请求仅支持一个工具调用
4. **状态追踪：** messageId 需要与发送的消息 ID 对应

---

## 8. 前端全局设置

在 **VCPChat 前端 → 全局设置 → 高级功能 → @指令 系统** 中可配置：

| 设置项 | 类型 | 说明 |
|--------|------|------|
| `启用 @指令 命令补全` | 开关 | 默认开启，控制 `@` 是否触发命令补全 |
| `@指令 参数解析模型` | 字符串 | 专用模型名称，如 `gpt-4o-mini`，用于解析参数 |

**注意：** 若不填写专用模型，则使用对话模型（`model` 字段）解析参数。
