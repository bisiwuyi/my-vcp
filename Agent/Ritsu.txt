**⚠️ 使用 OpenCode 进行代码工作时，必须严格遵守以下规定：**

### 📚 文档同步规定

在使用 OpenCode 对 VCP 系统进行开发时，必须同步创建和更新相关文档：

1. **公共文档库**：
   - 插件或服务介绍、使用方式 → `E:\VCP\VCPToolBox_new\dailynote\VCP百科全书\`
   - 开发相关文档 → `E:\VCP\VCPToolBox_new\docs\`

2. **插件文档**：
   - 每个插件的 README 文档必须及时创建和更新
   - 存放位置：`E:\VCP\VCPToolBox_new\Plugin\<插件名>\README.md`

3. **文档命名规范**：
   - 插件文档：`Plugin/<插件名>/README.md`
   - 百科条目：`dailynote/VCP百科全书/<功能名>.md`
   - 开发文档：`docs/<功能名>-development.md`

### 🔍 代码审查规定

**每次 OpenCode 完成代码任务后，必须执行两轮全面代码审查：**

### ✅ 验证测试规定

**OpenCode 完成任务后，必须进行验证测试：**

1. **功能验证**：确认代码按预期工作
2. **回归测试**：确认没有破坏现有功能
3. **异常测试**：确认边界条件和错误处理正确
4. **集成测试**：如果涉及系统集成，确认各组件协作正常

**测试通过标准**：
- 所有审查发现的问题都已解决
- 功能测试全部通过
- 无需重启系统即可验证的情况下，系统运行正常

**需要重启系统才能测试的情况**：
- 如果修改涉及服务器启动参数、端口绑定等
- 如果修改涉及插件加载机制
- 如果修改涉及数据库 schema 变更
在此情况下，记录需要重启验证的事项，告知用户重启后需再次测试
